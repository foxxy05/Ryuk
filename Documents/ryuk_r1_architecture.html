01 — Core Philosophy

Separation of Intent from Hardware

The central idea: a mechanism's state (what it should do) is completely separated from the motor ID (which physical actuator executes it). This means logic never knows about pins, timers, or addresses — and hardware never knows about strategy.

03 — The Router Pattern

How Hardware Gets Commanded

Inside each mechanism's .c file, a static hardware map binds Local Motor IDs to physical timer handles and channels. The Router function reads this map — the state machine never touches hardware directly.

STEP 1 — LOCAL HARDWARE MAP
// PRIVATE — invisible outside gripper.c
typedef struct {
  TIM_HandleTypeDef* timer;
  uint32_t channel;
  uint16_t home_pos;
  uint16_t active_pos;
} MotorHW_t;

static MotorHW_t g_hw[] = {
  [G_SERVO_0] = {&htim1, TIM_CHANNEL_1, 500, 2500},
  [G_SERVO_1] = {&htim1, TIM_CHANNEL_2, 500, 2500},
  [G_MTR_A] = {&htim3, TIM_CHANNEL_1, 0, 1000},
  // ... all gripper actuators
};
STEP 2 — ROUTER FUNCTION
// Local enum — never leaks outside
typedef enum {
  G_SERVO_0, G_SERVO_1, /* ... */
  G_MTR_A, G_MTR_B,
  G_TOTAL
} GripperID_t;

static void Gripper_Router(
  GripperState_t state,
  GripperID_t id
) {
  MotorHW_t *m = &g_hw[id];
  switch(state) {
    case ACT_ENGAGE:
      __HAL_TIM_SET_COMPARE(
        m->timer, m->channel,
        m->active_pos);
      break;
    case ACT_IDLE: /* home */ break;
  }
}
04 — Integrator View

What main.c Sees

From the integrator's perspective, the entire robot is four Init calls and four Update calls. No hardware details. No pin numbers. No timer handles.

// main.c — The entire robot in ~20 lines

/* Init Phase */
Gripper_Init();
Placer_Init();
Picker_Init();
Chassis_Init();

/* Main Loop — 100~200Hz, non-blocking */
while(1) {
  // 1. Read PS4
  PS4_Update(&ps4);

  // 2. Derive subsystem states from input
  GripperState_t gs = Map_Gripper(ps4);
  ChassisState_t cs = Map_Chassis(ps4);

  // 3. Execute — each handles its own hardware
  Gripper_Update(gs);
  Placer_Update(ps);
  Picker_Update(pks);
  Chassis_Update(cs);
}

06 — Competitive Advantages

The Pitch

🔌
Hot-Swap Hardware
Burnt motor, wrong pin? Change one line in g_hw[]. Zero logic touched. Done in 30 seconds pre-match.
⚡
Simultaneous Execution
Router loops all IDs in one pass. All 8 gripper servos fire in a single Update call. STM32 PWM hardware runs in parallel.
🔒
Encapsulation
Local enums per mechanism. Gripper cannot command Chassis motors — physically impossible. No accidental cross-talk.
🧪
Pick-and-Play Testing
Test Gripper alone by calling only Gripper_Init() + Gripper_Update(). No other subsystem involved. Isolated debugging.
🤝
Parallel Development
All 4 members code simultaneously. The contract is the header file. Internal implementation is entirely their own.
🛠️
2AM Debugging
State variable says ACT_ENGAGE but motor doesn't move? It's hardware/power, not code. Diagnosis in seconds, not hours.
